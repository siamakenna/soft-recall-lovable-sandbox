# Vertical Slice Scope

## Core Scenes

1. Bedroom / bedside
2. Mirror or phone close-up
3. Hallway / door

The kitchen, bathroom, living room, expanded checklist, and larger ending set are intentionally deprioritized until the first-person slice feels strong.

## Required Interactions

- Wake at bedside in a first-person view.
- Inspect glasses, phone, note, or voice memo.
- Turn to mirror or phone close-up.
- Answer a confidence prompt.
- Move into hallway.
- Inspect keys, appointment card, support note, and door.
- Use a support cue or phone support.
- Leave, pause, or make the morning smaller.

## Required Endings

- Supported Departure
- Smaller Morning
- Overloaded but Not Alone

## Intentionally Cut For Now

- Five-room progression
- Tea sequence
- Medication sequence
- Photo reconstruction chain
- Large dashboard layout
- Six-plus endings
- Heavy clinical/system explanation in README
- Audio, voiceover, sound effects, music, and generated audio

## Acceptance Criteria

- First playable screen reads as a first-person bedside scene.
- Main gameplay surface uses a 16:9 scene viewport.
- Floor plan is only a small notebook aid.
- VN box is the primary narrative interface.
- Keyboard/controller flow can complete core play.
- Object markers are small and embedded over visible objects.
- At least one ending is reachable in browser QA.
- No console errors, horizontal overflow, generic web-dashboard dominance, or overhead room map feel.
- No audio plays or is generated.
